# T01: Install Serwist and configure PWA with service worker, manifest, and Obsidian Deep theme

**Install Serwist and configure PWA with service worker, manifest, and Obsidian Deep theme**

## What Happened

Installed @serwist/next (v9.5.7) and serwist (v9.5.7) packages. Created service worker entry at src/app/sw.ts using defaultCache from @serwist/next/worker with skipWaiting, clientsClaim, and navigationPreload enabled. Wrapped next.config.ts with withSerwistInit() using swSrc: "src/app/sw.ts" and swDest: "public/sw.js". Created src/app/manifest.ts as a Next.js metadata route handler with Obsidian Deep colors (#0e0e10 background, #c0c1ff theme), standalone display mode, and references to /logo.png (512x512 maskable) and /logo.svg. Updated src/app/layout.tsx with PWA metadata (applicationName, appleWebApp, manifest link, icons) and separated themeColor into viewport export per Next.js 15 conventions. Added SerwistProvider to src/app/providers.tsx via dynamic import with ssr:false. The serwist core package was added as an explicit dependency because @serwist/next doesn't re-export its types directly — the initial build failed without it. After fixing, build succeeds with SW compilation step and all 498 tests pass.

## Verification

pnpm build succeeds with (serwist) Bundling the service worker script in output (exit 0). pnpm test passes with all 498 tests green (exit 0). grep -q 'withSerwist' next.config.ts confirms wrapper applied (exit 0). public/sw.js (43KB) generated by build. No build warnings after moving themeColor to viewport export.
