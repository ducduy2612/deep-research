## Directory Purposes

**`src/app/`:**
- Purpose: Next.js App Router pages and API routes
- Contains: Page components, API route handlers, middleware configuration
- Key files: `page.tsx`, `layout.tsx`, `not-found.tsx`, `sw.ts`, `middleware.ts`

**`src/app/api/`:**
- Purpose: Server-side API routes for external service proxies
- Contains: AI provider routes (`ai/*`), search provider routes (`search/*`), SSE endpoint, MCP server, crawler
- Key files: `sse/route.ts`, `mcp/server.ts`, `crawler/route.ts`

**`src/components/`:**
- Purpose: React components organized by feature
- Contains: Research workflow, knowledge management, settings, UI components
- Key files: `Research/`, `Knowledge/`, `Setting/`, `History.tsx`, `Artifact.tsx`

**`src/components/Internal/`:**
- Purpose: Internal application components
- Contains: Header, dialogs, inputs, debugger
- Key files: `Header.tsx`, `Setting.tsx`, `History.tsx`, `ModelInput.tsx`

**`src/components/ui/`:**
- Purpose: Shadcn/ui primitive components
- Contains: Button, dialog, form, input, etc.
- Generated by: shadcn/ui CLI

**`src/store/`:**
- Purpose: Zustand state management stores
- Contains: Global, Task, History, Knowledge, Setting stores
- Key files: `global.ts`, `task.ts`, `history.ts`, `knowledge.ts`, `setting.ts`

**`src/hooks/`:**
- Purpose: Custom React hooks for business logic
- Contains: Deep research orchestration, web search, knowledge management
- Key files: `useDeepResearch.ts`, `useWebSearch.ts`, `useKnowledge.ts`, `useAiProvider.ts`

**`src/utils/`:**
- Purpose: Utility functions and helper modules
- Contains: Deep research engine, search providers, parsers, file handling
- Key files: `deep-research/`, `parser/`, `storage.ts`, `model.ts`

**`src/utils/deep-research/`:**
- Purpose: Core research workflow implementation
- Contains: Research orchestration, prompt templates, provider factories
- Key files: `index.ts`, `provider.ts`, `search.ts`, `prompts.ts`

**`src/utils/parser/`:**
- Purpose: File parsing utilities for knowledge base
- Contains: PDF, Office, and text file parsers
- Key files: `pdfParser.ts`, `officeParser.ts`, `textParser.ts`

**`src/locales/`:**
- Purpose: Internationalization translation files
- Contains: JSON files for each supported language
- Key files: `en-US.json`, `zh-CN.json`, `es-ES.json`, `vi-VN.json`

**`src/constants/`:**
- Purpose: Application constants and configuration
- Contains: Prompt templates, URL constants, locale configurations
- Key files: `prompts.ts`, `urls.ts`, `locales.ts`

## Naming Conventions

**Files:**
- Components: PascalCase (e.g., `WorkflowProgress.tsx`, `FinalReport.tsx`)
- Utilities: camelCase (e.g., `deep-research/index.ts`, `file.ts`, `model.ts`)
- Hooks: camelCase with `use` prefix (e.g., `useDeepResearch.ts`, `useKnowledge.ts`)
- Stores: camelCase (e.g., `global.ts`, `task.ts`, `setting.ts`)
- Constants: camelCase (e.g., `prompts.ts`, `urls.ts`, `locales.ts`)

**Directories:**
- Features: PascalCase (e.g., `Research/`, `Knowledge/`, `Setting/`)
- Utilities: camelCase or lowercase (e.g., `deep-research/`, `parser/`)

**Components:**
- Feature components in feature directories: `src/components/Feature/ComponentName.tsx`
- Reusable components in feature index: `src/components/Feature/index.tsx`
- UI primitives in ui directory: `src/components/ui/component-name.tsx`

## Where to Add New Code

**New Feature:**
- Primary code: `src/components/FeatureName/`
- Tests: Not applicable (no test framework detected)
- Hooks: `src/hooks/useFeatureName.ts`

**New Component/Module:**
- Implementation: `src/components/FeatureName/ComponentName.tsx`
- UI primitives: `src/components/ui/component-name.tsx` (use shadcn CLI)

**Utilities:**
- Shared helpers: `src/utils/utilitiName.ts`
- Research-related: `src/utils/deep-research/featureName.ts`
- Parser-related: `src/utils/parser/fileTypeParser.ts`

**State Management:**
- New store: `src/store/featureName.ts`
- Follow pattern: Export store, interface, actions, default values

**API Routes:**
- AI provider: `src/app/api/ai/providerName/[...slug]/route.ts`
- Search provider: `src/app/api/search/providerName/[...slug]/route.ts`
- Other: `src/app/api/featureName/route.ts`
